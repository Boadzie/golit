Getting Started

We welcome contributions from everyone. Whether you're fixing a typo in the docs or designing a new scheduling engine, your help is valued.

check_circle Development Workflow

  1. 01. Fork the repository and create your feature branch from main.
  2. 02. Ensure all new code is accompanied by comprehensive unit tests.
  3. 03. Submit a Pull Request targeting the staging branch.

"Code is secondary; architecture is primary. Ensure your PR aligns with the DAG orchestration principles defined in the core specs."
